# あやかしとドラゴン — AYAKASHI & DRAGONS

[![CI](https://github.com/shou10152208/projector-action-rpg/actions/workflows/ci.yml/badge.svg)](https://github.com/shou10152208/projector-action-rpg/actions/workflows/ci.yml)

## 🎮 いますぐ遊ぶ

**→ https://shou10152208.github.io/projector-action-rpg/**

インストール不要。ブラウザで開いてカメラを許可するだけで遊べます（`https` 配信なのでスマホ・タブレットでもカメラOK）。
カメラが無ければそのままマウス／キーボードで遊べます。オフラインで遊びたい場合は下の[クイックスタート](#クイックスタート)へ。

> プロジェクターで壁に投影し、Webカメラの前で**全身を動かして**遊ぶ協力型アクションRPG（最大4人）。
> 和の妖怪と洋の魔物が入り混じる異界を旅し、レベルアップでスキルを選び、最終ボス**妖竜**を討伐せよ。

**手を振れば斬撃**、**拍手で衝撃波**、**両手を高く上げて必殺技**。
ロールによって固有アクションが変わります — 蹴りで戦う拳闘士、**手で円を描いて大火球**を放つ魔法使い、**縦一線の印で結界**を張る陰陽師、ジャスト回避からのカウンターが光る剣士。
カメラが無い環境では自動でマウス／キーボードに切り替わります。

## 遊びかた

1. **ロールを選ぶ**（手をかざして選択。全員が選ぶと出発）

   | ロール | 通常攻撃 | 固有アクション |
   | --- | --- | --- |
   | 剣士 | 腕を振って斬撃 | ジャスト回避で自動カウンター「燕返し」 |
   | 拳闘士 | 拳で殴る | **蹴り**も武器になる（足にも判定） |
   | 魔法使い | 魔弾（遠隔） | **手で円を描く**と大火球 |
   | 陰陽師 | お札投げ（遠隔） | **縦に一線を引く**と結界（弾を防ぎ敵を浄化） |

2. **マップで進む道を選ぶ** — 戦闘 / 強敵 / 宝箱 / 店 / 休憩 の分岐ルート。エリアクリア後は**次にゆく道**（豊穣・修羅・安息）も選べます
3. **戦って強くなる** — 敵を倒すとXPと**魂**（通貨）。レベルアップすると**全員同時に3択スキル**から自分の成長を選択（10秒で自動決定）
4. **共通アクション** — 拍手=衝撃波（2人同時なら**合体技**）／両手を胸の前で構えて**ガード**／攻撃の瞬間に体を素早く動かすと**ジャスト回避**（スローモー）／倒れた仲間は**手をかざして蘇生**
5. 和の霊峰 → 洋の魔城 → 混沌の狭間、3体のボス（大天狗・リッチ王・妖竜）を倒せば勝利！　1ラン約15分。

## クイックスタート

> **Python 3** が必要です（多くのPCに標準搭載）。Node.js やビルド作業は**不要**。
> MediaPipe 一式は同梱済みで**完全オフライン**で動きます。

### Windows
1. `start.bat` をダブルクリック
2. 既定のブラウザで自動的に開きます
3. カメラの使用許可を「許可」する

### macOS / Linux / WSL
```bash
./start.sh
```
（または `python3 server.py`）

開かない場合はブラウザで **http://localhost:8000/** を開いてください。

> **重要**: `index.html` をファイルとして直接開くと、ブラウザの制限でカメラとモジュール読込が動きません。
> 必ず上記サーバー経由（`http://localhost…`）で開いてください。

### マウス / キーボード操作（カメラなしでも遊べます）

| 操作 | 効果 |
| --- | --- |
| マウス移動 | 手カーソル |
| 左クリック / 長押し | 決定・攻撃 |
| `C` | 拍手（衝撃波） |
| `X` | ロール固有技 |
| 右クリック長押し / `Shift` | ガード |
| `Space` | 必殺技（ゲージ満タン時）／ 開始 |
| `M` 消音 / `D` デバッグ表示 / `F` 全画面 / `K` カメラ→マウス切替 / `G` カメラ再試行 | |

## 同じWiFiの別端末から遊ぶ（LAN）

サーバーは `0.0.0.0` で待ち受けるので、同一WiFiの別端末からアクセスできます。

- **マウス/タッチ操作**なら普通の **HTTP** でOK
- 別端末で**その端末のカメラを使う**には **HTTPS が必須**（ブラウザの仕様）→ `HTTPS=1 ./start.sh` で自己署名証明書つきで起動し、警告を承認して開く
- **WSL** の場合は Windows 側のポート転送が必要 → 管理者 PowerShell で `lan-setup.ps1` を実行

環境変数: `PORT=8080`（ポート変更）、`HTTPS=1`（TLS有効化）、`HOST=...`（既定 `0.0.0.0`）

## CI / CD

GitHub Actions（`.github/workflows/ci.yml`）が push / PR ごとに走ります。

- **syntax** — `node --check` で全JSモジュールを構文チェック
- **browser** — Playwright + ヘッドレスChromium で実走テスト。フェイクカメラでアプリを起動して戦闘まで実UIで進行し、さらにゲームロジックを直接動かして**勝利までのフル進行**（3エリア・3ボス・レベルアップ・ショップ）と単体検証を行う
- **deploy** — main が緑になったら **GitHub Pages へ自動デプロイ**

同じテストはローカルでも実行できます:
```bash
pip install playwright && python -m playwright install --with-deps chromium
python3 test/ci_test.py
```

> **Pages を有効にするには**（初回のみ）: リポジトリの **Settings → Pages → Source** を **GitHub Actions** に設定してください。
> Pages 配信ではレスポンスヘッダを設定できないため、同梱の `coi-serviceworker.js` が COOP/COEP を付与して WASM スレッド（認識の高速化）を有効にします。

## 技術メモ

- ビルド不要のバニラ JavaScript (ES Modules) + Canvas 2D + Web Audio（効果音は全てプロシージャル合成）
- 姿勢認識は MediaPipe PoseLandmarker（`vendor/` に同梱、完全オフライン）
- カメラ入力とマウス入力は同一の `player` オブジェクトに正規化され、ゲーム側は入力方式を意識しない
- チューニング値は `src/config.js` に集約
